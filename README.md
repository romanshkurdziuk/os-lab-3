# Лабораторная работа №3: Синхронизация потоков

## Описание

Проект представляет собой консольное приложение на C++, демонстрирующее принципы многопоточного программирования и синхронизации в среде Windows. Программа моделирует работу нескольких потоков-маркеров, которые параллельно заполняют общий массив целых чисел.

Основное внимание уделено решению проблем состояния гонки и обработке управляемых тупиковых ситуаций с использованием примитивов синхронизации WinAPI.

## Основные возможности

*   **Многопоточность:** Создание и управление несколькими рабочими потоками (`marker`).
*   **Синхронизация доступа:** Использование **критических секций** (`CRITICAL_SECTION`) для безопасного доступа к общему массиву и консольному выводу.
*   **Координация потоков:** Использование **событий** (`Event`) для реализации сложной логики:
    *   Одновременный старт всех рабочих потоков.
    *   Обнаружение ситуации, когда все потоки заблокированы.
    *   Управляемое возобновление и завершение работы потоков из главного потока (`main`).
*   **Современная система сборки:** Проект использует **CMake** для управления сборкой, зависимостями и структурой.
*   **Модульность:** Код декомпозирован на исполняемый модуль и статическую библиотеку `Utils`.
*   **Unit-тестирование:** Проект включает базовые модульные тесты для библиотеки `Utils` с использованием фреймворка **Google Test**.

## Как собрать и запустить

Проект использует CMake и требует наличия компилятора C++ (например, MinGW).

1.  **Клонировать репозиторий:**
    ```bash
    git clone https://github.com/romanshkurdziuk/os-lab-3.git
    cd os-lab-3
    ```

2.  **Сконфигурировать проект с помощью CMake:**
    ```bash
    mkdir build
    cd build
    cmake .. -G "MinGW Makefiles"
    ```

3.  **Собрать проект:**
    ```bash
    cmake --build .
    ```

4.  **Запустить основную программу:**
    ```bash
    ./src/os-lab-3.exe
    ```

5.  **Запустить тесты:**
    ```bash
    ctest --verbose
    ```
